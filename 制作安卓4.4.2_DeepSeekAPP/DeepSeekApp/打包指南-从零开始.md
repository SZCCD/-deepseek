# DeepSeek APP 从零开始打包指南

> 适用于全新电脑，无Android开发环境

---

## 方案一：使用 Android Studio（推荐）

### 步骤1：安装 Java JDK

**Windows:**
1. 访问 https://adoptium.net/
2. 下载 **OpenJDK 8 (LTS)** 或 **OpenJDK 11 (LTS)**
3. 运行安装程序，一路下一步
4. 安装完成后，打开命令提示符(cmd)，输入：
```cmd
java -version
```
如果显示版本信息，说明安装成功。

**Mac:**
```bash
brew install openjdk@11
```

**Linux (Ubuntu/Debian):**
```bash
sudo apt update
sudo apt install openjdk-11-jdk
```

---

### 步骤2：安装 Android Studio

1. 访问 https://developer.android.com/studio
2. 下载并安装 Android Studio
3. 首次启动时选择 **"Standard"** 安装
4. 等待下载完成（可能需要30分钟-1小时，取决于网速）

---

### 步骤3：配置 SDK

1. 打开 Android Studio
2. 点击菜单 `Tools` → `SDK Manager`
3. 在 **SDK Platforms** 标签页：
   - 勾选 **Android 9.0 (Pie) API 28**
   - 勾选 **Android 4.4 (KitKat) API 19**（可选，用于测试）
4. 在 **SDK Tools** 标签页：
   - 确保勾选 **Android SDK Build-Tools 28**
   - 确保勾选 **Android SDK Platform-Tools**
5. 点击 **Apply** 安装

---

### 步骤4：打开项目并打包

1. 启动 Android Studio
2. 选择 `Open an existing Android Studio project`
3. 选择 `DeepSeekApp` 文件夹
4. 等待 Gradle 同步完成（首次可能需要10-20分钟）
5. 同步完成后，点击菜单 `Build` → `Build Bundle(s) / APK(s)` → `Build APK(s)`
6. 构建完成后，右下角会显示通知，点击 `locate` 即可找到APK文件

**APK位置：** `app/build/outputs/apk/debug/app-debug.apk`

---

### 步骤5：生成签名APK（可选但推荐）

如果要发布给别人使用，需要签名：

1. `Build` → `Generate Signed Bundle / APK`
2. 选择 `APK`，点击 **Next**
3. 点击 **Create new...** 创建密钥：
   - Key store path: 选择一个位置保存（如 `C:\Users\你的用户名\deepseek-key.jks`）
   - Password: 设置密码（记住它！）
   - Key alias: `deepseek`
   - Key password: 设置密码
   - Validity: 25年
   - 填写姓名等信息（可以随便填）
4. 点击 **OK**，然后 **Next**
5. Build variant 选择 **release**
6. 勾选 **V1 (Jar Signature)** 和 **V2 (Full APK Signature)**
7. 点击 **Finish**

**签名APK位置：** `app/build/outputs/apk/release/app-release.apk`

## 方案二：命令行打包（适合高级用户）

### 步骤1：安装必要软件

**Windows:**

1. 安装 Chocolatey（包管理器）：
   - 以管理员身份打开 PowerShell
   - 运行：
```powershell
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
```

2. 安装 Java 和 Android SDK：
```powershell
choco install openjdk11
choco install android-sdk
```

3. 设置环境变量：
```powershell
[Environment]::SetEnvironmentVariable("ANDROID_HOME", "C:\Android\android-sdk", "User")
[Environment]::SetEnvironmentVariable("JAVA_HOME", "C:\Program Files\Eclipse Adoptium\jdk-11.0.20.101-hotspot", "User")
```

**Mac/Linux:**

```bash
# Mac
brew install openjdk@11
brew install android-sdk

# Linux (Ubuntu)
sudo apt install openjdk-11-jdk android-sdk
```

---

### 步骤2：下载项目并打包

```bash
# 进入项目目录
cd DeepSeekApp

# Windows 打包
gradlew.bat assembleDebug        # 调试版
gradlew.bat assembleRelease      # 发布版（需要签名配置）

# Mac/Linux 打包
./gradlew assembleDebug          # 调试版
./gradlew assembleRelease        # 发布版
```

## 方案三：在线打包（最简单，无需配置）

如果不想在本地配置环境，可以使用在线服务：

### 方法A：使用 GitHub Actions 自动构建

1. 将项目上传到 GitHub
2. 创建文件 `.github/workflows/build.yml`：

```yaml
name: Build APK

on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Set up JDK 11
      uses: actions/setup-java@v3
      with:
        java-version: '11'
        distribution: 'temurin'
    - name: Build with Gradle
      run: ./gradlew assembleDebug
    - name: Upload APK
      uses: actions/upload-artifact@v3
      with:
        name: app-debug
        path: app/build/outputs/apk/debug/app-debug.apk
```

3. 推送代码后，GitHub 会自动构建APK
4. 在 Actions 页面下载构建好的APK

### 方法B：使用 Appetize.io（在线模拟器）

1. 访问 https://appetize.io/
2. 上传APK文件可以在线预览
3. 但不提供打包功能

### 方法C：使用在线APK构建服务

一些第三方服务可以帮你构建：
- **Buildozer** (Kivy): https://buildozer.readthedocs.io/
- **PhoneGap Build** (已停止服务)

## 常见问题

### Q1: Gradle 下载很慢或失败？

**解决：** 修改 `gradle/wrapper/gradle-wrapper.properties`，使用国内镜像：

```properties
distributionUrl=https：//mirrors.cloud.tencent.com/gradle/gradle-5.6.4-all.zip
```

或在 `build.gradle` 中添加国内Maven仓库：

```gradle
allprojects {
    repositories {
        maven { url 'https：//maven.aliyun.com/repository/public' }
        maven { url 'https：//maven.aliyun.com/repository/google' }
        google()
        jcenter()
    }
}
```

### Q2: 提示 "SDK location not found"？

**解决：** 创建 `local.properties` 文件：

**Windows:**
```properties
sdk.dir=C\\:\\Users\\你的用户名\\AppData\\Local\\Android\\Sdk
```

**Mac:**
```properties
sdk.dir=/Users/你的用户名/Library/Android/sdk
```

**Linux:**
```properties
sdk.dir=/home/你的用户名/Android/Sdk
```

### Q3: 构建成功但安装失败？

可能原因：
1. **签名问题**：调试版APK可以直接安装，发布版需要签名
2. **设备不兼容**：确保设备Android版本 >= 4.4.2
3. **未知来源**：在设置中允许安装未知来源应用

### Q4: 安装后打开白屏？

1. 检查网络连接
2. 确保设备能访问 https://chat.deepseek.com/
3. 某些旧设备WebView版本太低，尝试更新系统WebView

## 快速检查清单

打包前确认：

- [ ] Java JDK 已安装 (`java -version` 有输出)
- [ ] Android Studio 已安装（方案一）或 Android SDK 已配置（方案二）
- [ ] 项目文件完整（特别是 `build.gradle` 和 `AndroidManifest.xml`）
- [ ] 网络连接正常（Gradle需要下载依赖）

## 推荐流程（最简单）

对于全新电脑，推荐按以下顺序：

1. **安装 Android Studio**（一步到位，包含所有必需组件）
2. **打开项目**，等待自动下载依赖
3. **点击 Build** → Build APK
4. **完成！**

预计总时间：1-2小时（主要取决于下载速度）
